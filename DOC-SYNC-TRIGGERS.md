# Doc-sync triggers

Canonical mapping from "this thing changed" to "these docs must be updated in lockstep." Extracted from `CLAUDE.md` so the hub doc stays under the 40k-char SessionStart threshold; the table is referenced from there as the source of truth.

A subset of these triggers are mechanically enforced by `scripts/check-doc-sync.mjs` (run via `pnpm doc-sync` and integrated into CI). It detects number-word drift between canonical sources of truth and target documents (edge type count, node type count, dashboard view count, public package count, workspace package list completeness). When adding a new "<word> <thing>" claim that should remain synchronized with an enumeration, extend the script with a new check to avoid reliance on manual review.

The non-mechanically-enforced rows below are discipline-only; the agent applying a change MUST consult this table before deciding the change is "done."

## Hub doc reference

`CLAUDE.md` keeps a short quick-reference of the most-frequent triggers and points here for the full set. When adding a new sync trigger, add it to this file (newest at the bottom of the relevant cluster) and consider whether the row warrants a slot in the CLAUDE.md quick-reference (criterion: ships in the repo's normal monthly cadence).

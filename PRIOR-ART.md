# Prior Art & Standards Map

Every specification, standard, and protocol that atrib builds on, extends, or hooks into. Organized by layer. Four relationship tags:

- **core:** atrib directly builds on this
- **extends:** atrib adds a layer on top of this
- **hooks:** atrib attaches to this at transaction time
- **context:** Relevant prior art or adjacent space

Draft 0.1, April 2026

## What No Existing Spec Provides

The gaps that atrib fills. None of the above specs address these:

| Gap                                         | Why it matters                                                                                                                                                                   |
| ------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Attribution record format**               | A signed, canonical data structure for recording that a specific creator's tool contributed to a specific session. OTel spans carry the data but are not signed by creators.     |
| **Value provenance graph**                  | A queryable graph connecting tool call events to transaction events via shared context, with typed edges (structural, co-occurrence, temporal). No existing spec defines this.   |
| **Attribution policy format**               | A machine-readable, negotiable document expressing how value should distribute across graph contributors. No existing spec defines weighting models for agent-mediated commerce. |
| **Settlement recommendation format**        | A verifiable output produced by applying a policy to a graph query ("Creator A gets 45%, Creator B gets 35%"), signed and auditable. Entirely novel.                            |
| **`atrib_context_id` in commerce metadata** | The convention for embedding an attribution context identifier in ACP/UCP checkout session metadata. Currently undefined in any commerce spec.                                   |
| **Cross-session recommendation token**      | A mechanism for attributing conversions that happen outside agent sessions (e.g., browser purchase after agent recommendation). Partially novel problem space.                   |

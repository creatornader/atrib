Layer 1 of the recall semantic surface lands as additive scaffolding. The existing `recall_my_attribution_history` tool gains new optional filter and ranking parameters: `min_importance` (annotation-derived importance threshold), `topic_tags` (OR-match against annotation topics), `include_revised` (hide records superseded by [D059](../DECISIONS.md#d059-promote-revision-to-atrib-normative-event_type-byte-0x06) revisions), `min_signers` (cross-attestation count filter), `rank_by` ('timestamp' | 'relevance' | 'causal_distance'), `rank_anchor` (anchor for non-timestamp ranking), and `toc` (table-of-contents response shape). The schema accepts annotation/revision in addition to the existing tool_call/transaction event_type short forms. Four new stub MCP tools register with full schemas and return a "Layer 1 in progress" notice: `recall_walk` (BFS over [§3.2.4](../atrib-spec.md#324-edge-derivation-rules) derived graph), `recall_annotations` ([D058](../DECISIONS.md#d058-promote-annotation-to-atrib-normative-event_type-byte-0x05) annotations pointing at a record), `recall_revisions` ([D059](../DECISIONS.md#d059-promote-revision-to-atrib-normative-event_type-byte-0x06) revision chain), and `recall_by_content` (free-form text search with BM25 fallback). The stubs publish the surface so downstream callers can wire against it now; full handler implementation lands in subsequent commits. Park et al. 2023 ranking weights and the importance scale are environment-tunable (ATRIB_RECALL_ALPHA / BETA / GAMMA / TAU_DAYS) for experiment-time per-axis sensitivity studies. Version bumps to 0.5.0-alpha; backward-compatible additive surface.

# atrib spec [§1.4](../../../atrib-spec.md#14-signing-and-verification) conformance corpus

Test vectors for signing, verification, and adversarial record inputs.

The corpus has two files:

| File                       | Asserts                                                                                                                                                                                      |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `signing-vectors.json`     | Deterministic public key derivation, canonical signing input, signature bytes, record hash, propagation token, and next `chain_root` for supported record shapes.                            |
| `adversarial-vectors.json` | Wycheproof-style negative and edge cases: bit-flipped signatures, truncated signatures, wrong creator keys, malformed context IDs, invalid event_type URIs, and JCS optional-field ordering. |

## Generators

`signing-vectors.json` is generated by the signing corpus generator used for Appendix A coverage.

`adversarial-vectors.json` is generated by:

```sh
pnpm --filter @atrib/log-dev exec tsx scripts/generate-conformance-1.4-adversarial.ts
```

Seeds and timestamps are fixed so regeneration is byte-identical unless the record format, canonicalization rule, or expected rejection behavior changes.

## Reference implementation

`packages/mcp/test/signing-corpus.test.ts` consumes `signing-vectors.json`.

`packages/mcp/test/signing-adversarial-corpus.test.ts` consumes `adversarial-vectors.json`.

Conforming implementations SHOULD load both files. The adversarial file is the offline CI floor; the live Wycheproof fetch in `packages/mcp/test/wycheproof.test.ts` remains an extra upstream compatibility check when network access is available.

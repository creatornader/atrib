# Evidence profile: `x401`

- **Type URI:** `https://atrib.dev/v1/evidence/x401`
- **Profile version:** `1.0.0` (semver of this document)
- **Status:** atrib-maintained. Mapped 1:1 from the legacy [§5.5.6](../../atrib-spec.md#556-generic-authorization-evidence-blocks) `x401` protocol string (frozen legacy set).

Carries x401 proof-of-payment / proof-gate authorization results for a signed action: the x401 header set, proof request/response, credential-verifier `resultVerified` output, and optional origin / trust / proof-payment-binding facts. x401 is authorization evidence, not payment detection ([D132](../../DECISIONS.md#d132-x401-proof-evidence-stays-verifier-side-authorization-evidence)); it never alters `verifyRecord().valid` ([§5.5.7](../../atrib-spec.md#557-universal-evidence-envelope)).

## Payload media types and hash rule

| Media type         | Payload                                   | Hash rule                   |
| ------------------ | ----------------------------------------- | --------------------------- |
| `application/json` | x401 header set / proof result / VC facts | JCS (RFC 8785, `jcsSha256`) |
| `application/jwt`  | VC / VP token (UTF-8)                      | raw bytes (`rawSha256`)     |

`payload.hash` = `"sha256:" + hex(SHA-256(bytes))`. Upstream Proof payloads and raw credential tokens stay hash-only ([D134](../../DECISIONS.md#d134-x401-producer-capture-and-propagation-stay-sanitized)).

## Facts schema

| Fact             | JSON type        | Provenance class |
| ---------------- | ---------------- | ---------------- |
| `issuer`         | string \| null   | verifier-derived |
| `subject`        | string \| null   | verifier-derived |
| `scope`          | string[]         | verifier-derived |
| `attenuation_ok` | boolean \| null  | verifier-derived |
| `delegation_ok`  | boolean \| null  | verifier-derived |
| `details_hash`   | string (sha256:) | verifier-derived |

`details_hash` commits the x401 proof-gate status, header source, result-by-reference, nonce binding, and any external verifier facts.

## Tier semantics

- `declared` / `shape` — asserted / offline-parsed x401 headers.
- `attested` — a caller-owned credential verifier returned `resultVerified` ([D132](../../DECISIONS.md#d132-x401-proof-evidence-stays-verifier-side-authorization-evidence)); legacy-mapping default.
- `verified` — VC/VP cryptographically verified and bound to the x401 nonce, reproducible from the envelope and declared trust roots.

## Verifier behavior

`verifyX401AuthorizationEvidence` in `@atrib/verify` produces the proof-gate status, header decode, and result constraints. Optional origin/trust/proof-payment-binding facts are surfaced when supplied. Failures set `result.valid: false` only.

## Sanitization contract

Public projections carry sanitized x401 facts and hashes only; raw Proof credentials, VP tokens, and upstream payloads MUST NOT appear ([D134](../../DECISIONS.md#d134-x401-producer-capture-and-propagation-stay-sanitized)).
